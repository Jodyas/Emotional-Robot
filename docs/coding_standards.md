# Coding Standards

1. **Explicit Typing**: Use `typing` module type hints consistently across inputs and return values.
2. **Docstrings**: Adopt Google-style docstrings for any major class, module, or function.
3. **Atomicity**: Git commits and functional units should resolve one concern exclusively.
4. **Resiliency**: Wrap external dependencies or potentially failing operations in clean try-except-finally blocks where practical.
